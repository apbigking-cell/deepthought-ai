# 多微信实例配置说明

## 功能
每个人格可以绑定独立的微信号，实现"一个角色一个微信号"的效果。
朋友加不同微信号，直接和对应人格聊天，无需切换命令。

## 配置方法

在 Railway 的 Variables（或本地 .env）里添加：

### 主微信（默认路由，所有用户共享）
```
WEIXIN_BOT_TOKEN=主微信的token
```

### 芙宁娜专属微信
```
WEIXIN_BOT_2_TOKEN=芙宁娜微信的token
WEIXIN_BOT_2_PERSONA=furina
WEIXIN_BOT_2_NAME=芙宁娜专属
```

### 小语专属微信（示例）
```
WEIXIN_BOT_3_TOKEN=小语微信的token
WEIXIN_BOT_3_PERSONA=xiaoyu
WEIXIN_BOT_3_NAME=小语专属
```

### 林夏专属微信（示例）
```
WEIXIN_BOT_4_TOKEN=林夏微信的token
WEIXIN_BOT_4_PERSONA=linxia
WEIXIN_BOT_4_NAME=林夏专属
```

## 获取 Token 的方法

每个微信号都要单独扫码获取 token：

1. 在 Railway 控制台运行：
   ```bash
   node -e "import('./src/bot/weixin.js').then(m => m.weixinLoginInteractive().then(r => console.log(r)))"
   ```
2. 终端会显示二维码链接
3. 用对应的微信号扫码
4. 复制返回的 token
5. 粘贴到 Railway Variables

## 注意事项

1. **每个微信号必须独立**——不能两个 Bot 用同一个 token
2. **扫码时本地不要同时运行**——会冲突
3. **Railway 免费额度限制**——多实例会多消耗内存，建议最多 2-3 个
4. **绑定后无法用 /人格 命令切换**——绑定的 Bot 强制使用对应人格

## 工作机制

- 主微信（WEIXIN_BOT_TOKEN）：所有用户共享，可用 /人格 命令切换
- 附加微信（WEIXIN_BOT_2_TOKEN 等）：绑定后强制路由到指定人格，无法切换
- Web 聊天：仍然可以在面板上自由切换所有人格
